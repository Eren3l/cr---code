# 03 - 数据库设计

> 项目：简生 JianSheng 健身打卡微信小程序
> 版本：v1.0 MVP
> 更新日期：2026-07-04

---

## 概述

- 数据库类型：微信云数据库（MongoDB 风格文档数据库）
- 共 5 个集合（Collection）
- 每条记录自动包含 `_id`（系统生成）和 `_openid`（用户标识，由云开发自动管理）

---

## 1. workouts — 运动打卡记录

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `_openid` | String | 自动 | 用户标识（云开发自动添加） |
| `date` | String | ✅ | 打卡日期 "YYYY-MM-DD" |
| `type` | String | ✅ | 运动类型 |
| `bodyPart` | String | | 训练部位 |
| `sets` | Number | | 组数 |
| `reps` | Number | | 次数 |
| `weight` | Number | | 重量(kg) |
| `duration` | Number | | 时长(分钟) |
| `distance` | Number | | 距离(km) |
| `pace` | String | | 配速 |
| `heartRate` | Number | | 平均心率 |
| `feeling` | String | | 感受：轻松/适中/吃力/力竭 |
| `calories` | Number | | 估算消耗(kcal) |
| `photos` | Array | | 云存储文件ID数组 |
| `note` | String | | 备注 |
| `isMakeup` | Boolean | | 是否为补卡，默认 false |
| `createdAt` | Date | ✅ | 创建时间 |

**索引建议**：`date` + `_openid` 复合索引（按日期查询最频繁）

## 权限设置

所有集合统一设置为：**仅创建者可读写**

在云开发控制台 → 数据库 → 集合 → 权限设置中配置。

## 日历数据说明

日历无需单独建表，直接查询 `workouts` 和 `diets`：
- 某天有运动记录 **或** 饮食记录 → 🟢 已打卡
- 某天无记录 → ⚪ 未打卡
- 昨天有记录但今天无记录 → 🔴 断卡（连续中断）
