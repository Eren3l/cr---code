# 06 - 开发执行步骤

> 项目：简生 JianSheng 健身打卡微信小程序
> 版本：v1.0 MVP
> 更新日期：2026-07-04

---

## 核心原则

1. **每阶段只做一件事**，完成 → 验证 → 提交 → 下一阶段
2. **真机测试贯穿始终**，不要只在模拟器里看
3. **每完成一个子步骤提交一次 Git**，出问题可随时回退
4. **代码宁可简单、不要炫技**——这是给新手维护的项目

---

## 第 0 阶段：环境搭建（预计 1-2 天）

**目标：** 让小程序能在手机上跑起来

### 步骤

1. **注册微信小程序账号**
   - 访问 mp.weixin.qq.com，使用微信扫码注册
   - 选择「小程序」类型
   - 填写个人信息，完成实名认证
   - 在「开发管理」→「开发设置」中获取 AppID

2. **下载安装微信开发者工具**
   - 访问 developers.weixin.qq.com/miniprogram/dev/devtools/download.html
   - 下载 Windows 64 位版本
   - 安装并登录

3. **创建小程序项目**
   - 打开开发者工具，点击「+」创建新项目
   - 项目名称：简生
   - 目录：选择 `e:\VC\JianSheng`
   - AppID：填入第一步获取的 AppID
   - 开发模式：小程序
   - 模板选择：**微信云开发**（重要！）
   - 点击确定

4. **开通云开发**
   - 在开发者工具顶部菜单点击「云开发」
   - 点击「开通」，创建一个新环境
   - 环境名称：`jiansheng`（或自定义）
   - 开通完成后，记录环境 ID（类似 `jiansheng-xxxxx`）

5. **验证**
   - 在开发者工具中点击「预览」
   - 手机微信扫码，确认能看到小程序首页

### 验证清单
- [ ] 已拿到 AppID
- [ ] 开发者工具能正常打开项目
- [ ] 云开发已开通，能看到云开发控制台
- [ ] 手机扫码能打开小程序

## 第 1 阶段：底部导航 + 四个空页面（预计 1 天）

**目标：** 搭好小程序骨架，四个 Tab 能切换

### 步骤

1. 配置 `app.json` tabBar（4个Tab，主色调 #4A90D9）
2. 创建四个页面文件夹及文件（.js / .wxml / .wxss / .json）
3. 每个页面写简单占位文字
4. 准备 Tab 图标（8张PNG：每个Tab选中+未选中）
5. 设置全局样式 `app.wxss`（背景色 #F5F6FA，主色调，字体）
6. 真机预览测试
7. Git 提交

### 验证清单
- [ ] 四个 Tab 能正常切换
- [ ] 选中态颜色为淡蓝色 #4A90D9
- [ ] 未选中态为灰色 #999999
- [ ] 页面背景为浅灰白 #F5F6FA

---

## 第 2 阶段：云数据库搭建（预计 1 天）

**目标：** 在云开发控制台创建好所有数据表

### 步骤

1. 打开云开发控制台 → 数据库
2. 创建 5 个集合：`workouts`、`diets`、`water_records`、`templates`、`goals`
3. 为每个集合设置权限：仅创建者可读写
4. 手动插入 1-2 条测试数据
5. 写一段简单代码在 `app.js` 中初始化云开发
6. 在任一页面中测试读写数据库
7. Git 提交

### 验证清单
- [ ] 5 个集合全部创建成功
- [ ] 权限设置正确
- [ ] 能在小程序中成功写入和读取数据

## 第 3 阶段：运动打卡页（预计 3-4 天）

**目标：** 运动打卡功能完整可用

### 子步骤

**3.1 运动类型选择器**
- 预设 8 种运动类型（用标签/tag展示）
- 支持输入自定义运动类型
- 点击选中，高亮显示

**3.2 详细打卡表单**
- 所有字段可选填
- 表单字段：组数、次数、重量、时长、距离、配速、心率
- 训练部位多选标签
- 感受单选（轻松/适中/吃力/力竭）

**3.3 照片上传 + 云存储**
- 调用微信选择图片 API
- 上传到云存储
- 图片上叠加日期+运动类型文字水印（用 Canvas）

**3.4 训练模板系统**
- 模板列表（横向滑动）
- 从当前填写内容新建模板
- 点击模板一键填充表单

**3.5 历史记录**
- 当天已打卡记录列表
- 支持删除

**3.6 提交打卡**
- 数据校验
- 写入云数据库
- 成功提示

### 验证清单
- [ ] 完整走通：选类型 → 填数据 → 传照片 → 打卡 → 数据入库
- [ ] 模板创建和选择正常
- [ ] 照片水印正确显示

## 第 4 阶段：饮食打卡页（预计 2-3 天）

**目标：** 饮食打卡功能完整可用

### 子步骤

**4.1 四餐分区 UI**
- 早餐/午餐/晚餐/零食四个卡片区域
- 每个区域显示已记录食物列表 + 添加按钮

**4.2 添加食物弹窗**
- 餐别选择（自动填入当前区域餐别）
- 食物名称输入
- 热量输入
- 备注（可选）
- 拍照（可选）

**4.3 食物记录列表**
- 每条记录显示：名称 + 热量
- 左滑删除

**4.4 饮水记录**
- 杯子图标网格（8杯）
- 点击+1杯
- 已喝/目标 毫升数显示
- 可设置单杯容量（默认 250ml）

**4.5 热量统计**
- 今日四餐合计热量
- 进度条对比目标

### 验证清单
- [ ] 四餐分区各自独立添加和显示
- [ ] 左滑删除正常
- [ ] 饮水点击计数正确
- [ ] 热量合计准确

## 第 5 阶段：打卡日历页（预计 2-3 天）

**目标：** 日历可视化和目标管理完整可用

### 子步骤

**5.1 日历组件**
- 月视图渲染（7列 x 6行网格）
- 月份前后切换
- 今天特殊标记（蓝色圆圈）

**5.2 颜色标记**
- 查询当月所有打卡记录
- 有打卡 → 绿色背景
- 无打卡 → 灰色
- 断卡 → 红色标记

**5.3 日期点击**
- 点击某天查看当天打卡详情
- 弹窗或跳转展示

**5.4 月度统计**
- 训练天数
- 饮食记录天数
- 连续打卡天数
- 当月总训练次数

**5.5 目标管理**
- 当前目标列表
- 进度条可视化
- 新建/编辑/删除目标

**5.6 补卡功能**
- 选择历史日期
- 填写打卡内容
- 标记 isMakeup = true
- 填写断卡原因

### 验证清单
- [ ] 日历正确渲染当月日期
- [ ] 颜色标记与数据库记录一致
- [ ] 补卡后日历颜色更新
- [ ] 目标进度计算正确

## 第 7 阶段：联调 + 优化（预计 2 天）

**目标：** 全局体验打磨

### 步骤

1. 连续打卡天数计算逻辑（跨天检测）
2. 断卡检测逻辑
3. 热量估算公式（运动类型 × 时长 × 体重系数）
4. 各页面数据联动（运动打卡后日历同步更新）
5. UI 细节调整
6. 异常处理（网络断开提示、数据库错误友好提示）
7. 整体走查测试

## 第 8 阶段：提交审核上线（预计 1-2 天）

**目标：** 小程序发布上线

### 步骤

1. 在 mp.weixin.qq.com 填写小程序基本信息
2. 设置类目（建议：工具 → 健康）
3. 在开发者工具中上传代码

5. 审核通过后发布
6. 生成小程序码，开始使用
