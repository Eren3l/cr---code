# CLAUDE.md — 简生 JianSheng 项目 AI 助手指令

## 项目概况

**简生 (JianSheng)** 是一个面向个人使用的健身打卡微信小程序。

- **平台**：微信小程序（手机端，在 Windows 上用微信开发者工具开发）
- **后端**：微信云开发（云数据库 + 云存储 + 云函数）
- **用户**：完全不懂代码的新手，需要手把手级别的开发教程
- **UI**：简洁直观，主色调淡蓝色 `#4A90D9`

## 工作流程

### 当用户发出开发指令时

1. **先读标准文档** — 确认用户需求是否在 MVP 范围内（参考 `01-需求说明.md`）
2. **解释要做什么** — 用通俗的语言告诉用户接下来要进行什么操作
3. **分步执行** — 每完成一个小步骤停下来让用户确认
4. **每步验证** — 告诉用户如何验证这步做对了
5. **记录日志** — 完成一个阶段后在 `dev-logs/` 下创建当天的日志文件

### 当用户遇到报错时

1. 先让用户截图或复制错误信息
2. 用通俗语言解释错误原因
3. 给出具体的修复步骤
4. 修复后验证

### 当用户提出新功能需求时

1. 查 `docs/01-需求说明.md` 确认是否在规划范围内
2. 如果在 V2.0/V3.0 范围，提醒用户当前是 V1.0 MVP，建议先完成基础版本
3. 如果用户坚持，评估影响并更新需求文档

## 开发规范

### 代码风格
- 使用微信小程序原生框架，不引入第三方 UI 库
- JavaScript 使用 ES6 语法
- 所有用户可见文字用中文
- 变量和函数命名用英文驼峰（camelCase）
- 注释用中文

### Git 提交规范
- 每完成一个子步骤提交一次
- 提交信息用中文：`[阶段N.M] 做了什么`
- 示例：`[3.1] 完成运动类型选择器`

### 文件组织
```
miniprogram/
├── pages/           # 页面（每个页面一个文件夹）
├── components/      # 公共组件
├── images/          # 图片和图标
├── utils/           # 工具函数
├── app.js           # 入口
├── app.json         # 全局配置
└── app.wxss         # 全局样式
```

## 对 AI 的要求

- **默认语言**：中文
- **解释方式**：假设用户完全不懂编程，用类比和比喻解释技术概念
- **操作粒度**：每步操作描述要具体到"点击哪里""输入什么"，不要省略细节
- **安全优先**：涉及数据库、云函数权限的配置，必须明确说明权限设置
- **避免过度设计**：V1.0 只做 MVP，不要主动提议加功能
